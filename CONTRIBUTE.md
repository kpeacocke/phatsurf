
# Contributing to Surfboard Recommender and Alerts App

Thank you for your interest in contributing to this project! We welcome contributions that help improve the app. Please read the following guidelines to ensure a smooth collaboration process.

## How to Contribute

1. **Fork the Repository**
   - Fork this repository and clone your fork locally.
   - Set up the upstream remote:
     ```bash
     git remote add upstream https://github.com/yourusername/repo-name.git
     ```

2. **Create a Branch**
   - Create a new branch for your contribution:
     ```bash
     git checkout -b feature/your-feature-name
     ```

3. **Make Your Changes**
   - Follow the existing code style and project structure.
   - Test your changes to ensure they work as expected.

4. **Commit Your Changes**
   - Write clear and concise commit messages:
     ```bash
     git commit -m "Add: Your descriptive commit message"
     ```

5. **Push Your Changes**
   - Push your branch to your fork:
     ```bash
     git push origin feature/your-feature-name
     ```

6. **Submit a Pull Request**
   - Open a pull request (PR) to the `main` branch of this repository.
   - Provide a detailed description of your changes and the problem they solve.

## Guidelines

- Ensure your code is well-documented.
- Add tests for new features or bug fixes where applicable.
- Follow the [Code of Conduct](#code-of-conduct).

## Code of Conduct

All contributors are expected to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md). Please treat others with respect and professionalism.
